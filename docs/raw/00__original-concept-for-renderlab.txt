== Original Prompt ==
date: 2026-03-26

I want to design an AI Studio-like (Google's "AI Studio") system.

AI Studio has this thing where -- when you go to their page -- if you submit a a bunch of elements for a prompt -- some files: JSON, text files, image files, -- and set it up as a prompt that you're about to submit, and then hit this button "turn to code" (or something like that,) it'll actually generate Python code for submitting that prompt, including the file submissions and everything -- which: is awesome.  I absolutely love that.

But we need to (and "we" here is Robert + I -- and I suppose I'm shang-hai'ing you into our adventures here,) -- we need to make something where: We can perform six requests in parallel.  Any number of requests, in parallel.  And prepare requests in parallel.  You know, if I could set up a request, and then duplicate it six times, and make a slight variation in each one, and kick them all off to make 4 renders each -- it'd be amazing.  And then have tracking and records straight for everything.

We talked about this earlier, I just haven't been able to find the prior conversation.  The idea was that you would build the request details in a folder, including all tracking details, and then for each rendering, store the results in a file in a "renders" sub-directory.

Sometimes I have a prompt divided into two JSON files.  One part varies, and another part stays the same.  Like one represents a solid brick of code, and the other represents parameters to it.  "Render this design...  Now this one...  And now this one."  One file is the same across all of them, and the other file is differing.  Oh, and we have a Hanko in an image file that we attach to each image, as well, -- or a signature, or something.

I want this all to be easy, automatic.  I also -- want to make it incrementally. I don't want to program the whole thing at once. I'd like to be able to preview the, or review the things that are output. I want to be able to run it one time or six times, create multiple images in a single go. I'd prefer that it be tkinter-based rather than CLI-based or Python import based. Although, I mean, it obviously doesn't hurt to have those as well. But -- CLI first.  Viewing outputs, changing prompts.

Also, eventually I would like to grow it into something where I can take an image, use green to block out part of that image and then say, hey, can you fill this? Because there often is just something broken that needs to be fixed.

I have had thoughts about -- how to make it part of a FileTalk ecology.  I think the thing to do is have an images directory, then, that is shared by the various programs.  Put images in there with a GUID as the key part of the file identifier (the file name), and pass the pointer to it in the filetalk messages.  Periodically, just delete everything in there older than say 4 hours.  (It's essentially a temporary store for parameter-like passed images.  Things that output a file then copy it to a relatively-speaking permanent location.)

Thoughts on all of this?
